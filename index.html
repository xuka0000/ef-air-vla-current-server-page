Abstract

This page summarizes the current Work4 RTX5090 server evidence for AIR-VLA and the proposed Execution-Feedback AIR-VLA direction. The server can run D0-D1 smoke checks over the available AIR-VLA HDF5 subset. The HDF5 audit passed for 20 files, the HDF5-only CUDA BC smoke passed over all 20 episodes, and the EF-AIR-VLA CUDA smoke passed over the two real feedback rows currently available.

The same sweep shows why complete reproduction is not ready. Public assets still lack training code, a policy inference server, and checkpoint weights. The current dataset has 20 episodes and one evaluator task. The official original evaluator returned code -11 and produced no rollout artifacts. The page therefore reports server readiness and reproduction friction only.

Method Overview

The page follows a paper-project layout while keeping every claim tied to evidence paths and truth levels.

1

Subset Audit

Open and check the 20 available AIR-VLA HDF5 files before any learning smoke.

2

HDF5-only BC Smoke

Train a small CUDA model from state means to action means over all 20 HDF5 episodes.

3

Feedback Smoke

Join state means with execution-feedback fields and test checkpoint, resume, and inference.

4

Reproduction Gates

Block long training until public assets, dataset coverage, evaluator stability, and checkpoints exist.
