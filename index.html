Abstract

This page records the current Work4 server evidence for AIR-VLA reproduction and Execution-Feedback AIR-VLA design. HDF5 audits, CUDA smokes, evaluator diagnostics, and D1 to D7 candidate runs are linked to local artifacts.

The current result is an engineering diagnostic rather than a performance claim. Official training code, policy inference assets, checkpoints, broad data coverage, and a stable full evaluator are still missing.

Method Overview

The page follows a paper-project layout while keeping every claim tied to evidence paths and truth levels.

1

Subset Audit

Open and check the 20 available AIR-VLA HDF5 files before any learning smoke.

2

HDF5-only BC Smoke

Train a small CUDA model from state means to action means over all 20 HDF5 episodes.

3

Feedback Smoke

Join state means with execution-feedback fields and test checkpoint, resume, and inference.

4

Reproduction Gates

Block long training until public assets, dataset coverage, evaluator stability, and checkpoints exist.

Evidence And Boundaries

Supported by this page

  • Current server can run CUDA smoke checks for HDF5-only BC and EF feedback plumbing.
  • The available HDF5 subset is valid for D0 checks, with 20 files and one task.
  • Full reproduction is blocked by missing assets, insufficient data coverage, and official evaluator instability.

Not supported

  • No AIR-VLA+ full reproduction.
  • No official AIR-VLA training result.
  • No learned-policy task-completion claim.
  • No hardware validation or sim-to-real claim.
